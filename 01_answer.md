Software Development Life Cycle (SDLC) adalah proses terstruktur yang digunakan untuk merencanakan, merancang, mengembangkan, menguji, melakukan deployment, dan memelihara sistem perangkat lunak secara efisien dan berkualitas.

1. Requirement analysis: QA memastikan requirement jelas, lengkap, dan bisa diuji.
   Mengidentifikasi kebutuhan yang ambigu serta menentukan acceptance criteria dan strategi testing awal.
   Tujuannya untuk mencegah defect sebelum development dimulai.

2. Design phase: QA mereview desain sistem dan arsitektur.
   Mengidentifikasi area berisiko serta memastikan sistem mudah diuji.
   Tujuannya untuk memastikan desain mendukung proses testing.

3. Development phase: QA membuat test case dan test data serta menyiapkan automation jika diperlukan.
   Melakukan early testing dan koordinasi dengan developer.
   Tujuannya untuk menemukan bug lebih awal.

4. Testing phase: QA menjalankan testing (functional, integration, UAT), mencatat defect, dan memverifikasi perbaikan bug.
   Tujuannya memastikan software sesuai kebutuhan dan berkualitas.

5. Deployment phase: QA melakukan smoke testing dan validasi setelah rilis ke production.
   Tujuannya untuk memastikan aplikasi berjalan stabil saat release.

6. Maintenance phase: QA melakukan regression testing, validasi update, dan monitoring issue berdasarkan feedback user.
   Tujuannya untuk menjaga kualitas sistem setelah digunakan.
